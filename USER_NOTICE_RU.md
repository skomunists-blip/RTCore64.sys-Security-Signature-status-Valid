# RTCore64.sys: кратко для пользователей

Этот репозиторий содержит исследование безопасности драйвера `RTCore64.sys`.

## Что это значит простыми словами

`RTCore64.sys` работает в ядре Windows (Ring 0). Такие драйверы имеют очень высокий уровень доступа к системе. Если драйвер открывает опасные операции для пользовательских программ, даже программа с правами администратора может получить доступ к тому, к чему обычное приложение обращаться не должно.

В рамках исследования подтверждено:

- драйвер открывает опасные управляющие ветки через `DeviceIoControl`;
- есть ошибка порядка проверок в unmap-пути;
- есть слабая проверка границ в read/write-пути;
- есть подтвержденный crash/DoS-класс;
- есть доступ на чтение/запись к некоторым MMIO/VRAM-окнам видеоустройства;
- возможно раскрытие видимого framebuffer/VRAM-контента.

Не подтверждено:

- чтение всей RAM;
- запись в произвольную RAM;
- прямое повышение прав от обычного пользователя;
- выполнение команд на GPU.

## Почему это важно

Даже если полный доступ ко всей RAM не доказан, драйвер все равно является опасной поверхностью:

- работает в ядре;
- имеет историю публичных злоупотреблений в классе BYOVD;
- сохраняет небезопасные паттерны;
- позволяет делать операции, которые обычным приложениям не должны быть доступны.

## Что делать пользователям

Рекомендации:

- не держать старые или ненужные версии MSI Afterburner / RTSS / RTCore64;
- включить Microsoft Vulnerable Driver Blocklist;
- включить Core Isolation / Memory Integrity, если совместимо с вашей системой;
- не запускать неизвестные программы с правами администратора;
- проверить, загружен ли `RTCore64.sys`, если вы не используете MSI Afterburner.

Проверка в PowerShell:

```powershell
Get-CimInstance Win32_SystemDriver | Where-Object { $_.Name -match 'RTCore' } |
  Select-Object Name, State, PathName
```

Если драйвер не нужен, лучше удалить программу, которая его установила, штатным способом.

## Важная граница

Этот проект не публикует эксплойт и не содержит драйверный бинарь. Цель проекта - defensive security research и помощь разработчикам/пользователям в оценке риска.
